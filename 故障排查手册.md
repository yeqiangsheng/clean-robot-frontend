# 故障排查手册

## 1. 前端启动后直接显示错误页

先看页面上的启动阻断信息。
当前版本会在业务页挂载前校验 [public/app-config.json](/c:/work/clean-robot-frontend/public/app-config.json)。

重点检查：

- `rosbridgeUrl` 是否为合法 `ws://` 或 `wss://`
- `quickRosbridgeUrls` 是否为非空数组
- `engineerUnlockMode` 是否为 `direct`
- 字段是否缺失或类型错误

修复后刷新页面即可。

## 2. 生产脚本启动失败

先执行：

```powershell
npm.cmd run verify
```

再检查：

- `.tmp/frontend-prod/frontend.out.log`
- `.tmp/frontend-prod/frontend.err.log`

常见原因：

- 依赖未安装
- `verify` 未通过
- `dist` 未生成
- 端口 `4173` 已被其他进程占用

## 3. 页面能打开，但 ROS 一直未连接

检查：

- `public/app-config.json` 中的 `rosbridgeUrl`
- rosbridge 进程是否已启动
- 本机防火墙是否拦截 WebSocket 端口
- 目标地址是否能从现场机器访问

建议先用总览页确认：

- 当前 endpoint
- 连接状态
- 能力探测结果

## 4. Mock 可以，Live 不可以

这通常说明前端壳层没问题，故障在现场 ROS 依赖或配置。

建议顺序：

1. 先跑 mock smoke，确认页面本身正常。
2. 再检查 rosbridge 连接。
3. 再看能力探测中缺失的 service 或 topic。
4. 最后导出诊断包交给研发分析。

## 5. SLAM 或执行机构页报错

先区分两类问题：

- 页面权限阻断
- ROS 依赖缺失或未连接

当前版本中：

- 高风险功能需要工程师模式
- 高风险动作统一经 gateway 下发并写审计
- mock 模式下页面应可打开，但不会验证真实 ROS 依赖

## 6. 如何导出诊断包

进入总览页，点击 `Export Diagnostics`。

导出内容包含：

- 构建版本
- 脱敏配置快照
- 当前 rosbridge 连接状态
- 能力探测结果
- 最近审计记录
- runtime topic 健康度
- 最近错误

## 7. 如何安全停止前端

优先使用：

```powershell
.\stop-frontend-prod.cmd
```

不要直接杀掉未知 PID。
如果脚本提示端口不是 `node` 进程占用，需要先确认现场是否还有其他服务共用该端口。
