# 现场验收清单

## P0 基线

- [ ] `npm.cmd run lint` 通过
- [ ] `npm.cmd run typecheck` 通过
- [ ] `npm.cmd run build` 通过
- [ ] `npm.cmd run verify` 通过

## 生产启动

- [ ] `.\start-frontend-prod.cmd` 可成功拉起静态服务
- [ ] `http://127.0.0.1:4173` 健康检查返回 `200`
- [ ] `.tmp/frontend-prod/frontend.pid` 已生成
- [ ] `.tmp/frontend-prod/frontend.out.log` 有启动日志
- [ ] `.\stop-frontend-prod.cmd` 可正常停止服务

## Mock 验收

- [ ] `npm.cmd run test:e2e` 通过
- [ ] 总览页可打开
- [ ] 地图页可打开
- [ ] 任务页可打开
- [ ] 调度页可打开
- [ ] 执行控制页可打开
- [ ] 运行监控页可打开
- [ ] 工程师模式开启后，SLAM 页可打开
- [ ] 工程师模式开启后，执行机构调试页可打开
- [ ] “Export Diagnostics” 可导出诊断包

## Live 验收

- [ ] rosbridge 可连接
- [ ] rosbridge 可断开后重连
- [ ] 任务 CRUD 有明确成功/失败反馈
- [ ] 调度 CRUD 有明确成功/失败反馈
- [ ] 执行控制可给出成功/失败/阻断反馈
- [ ] SLAM submit 和 job 轮询有明确反馈
- [ ] 执行机构调试动作有明确反馈
- [ ] 高风险动作会写入本地审计记录

## 异常验收

- [ ] 错误 `app-config.json` 会阻断启动并显示修复建议
- [ ] 缺少 topic 或 service 时页面会显示降级/缺失状态
- [ ] 工程师模式不依赖前端明文口令

## 现场留档

- [ ] 启动成功页截图
- [ ] 总览页截图
- [ ] rosbridge 已连接截图
- [ ] 任务/调度联调结果截图
- [ ] SLAM 或执行机构高风险动作反馈截图
- [ ] 诊断包导出文件留存
- [ ] `frontend.out.log` 与 `frontend.err.log` 留存
