# 现场验收清单

## 1. 发布包基线

- [ ] 已执行 `npm.cmd run package:trial`
- [ ] 已阅读 `docs/commercial_release_readiness_0.1.0-rc.2.md`
- [ ] 发布目录包含 `dist/`
- [ ] 发布目录包含 `public/app-config.json`
- [ ] 发布目录包含 `site-gateway/`
- [ ] 发布目录包含 `scripts/`
- [ ] 发布目录包含 `RELEASE-INFO.json`

## 2. 安装与启动

- [ ] 在发布目录执行 `npm.cmd install --omit=dev` 成功
- [ ] `.\start-frontend-prod.cmd` 可以成功拉起站点网关
- [ ] `http://127.0.0.1:4173` 可以打开
- [ ] `http://127.0.0.1:4173/api/health` 返回 `200`
- [ ] `.tmp/frontend-prod/frontend.pid` 已生成
- [ ] `.tmp/frontend-prod/frontend.out.log` 有启动日志
- [ ] `.\stop-frontend-prod.cmd` 可以正常停止服务

## 3. 服务化验收

- [ ] `.\scripts\install-site-service.ps1 -WinSwExePath <path>` 可以成功安装服务
- [ ] Windows 服务 `CleanRobotSiteGateway` 状态正常
- [ ] 服务启动后浏览器入口可访问
- [ ] `.\scripts\uninstall-site-service.ps1` 可以正常卸载服务

## 3A. Ubuntu 小车触摸屏 kiosk 验收

- [ ] 小车本体发布包安装目录为 `/opt/clean-robot-site/current` 或现场约定目录
- [ ] `npm install --omit=dev` 在小车发布目录执行成功
- [ ] `clean-robot-site-gateway` systemd 服务已启用并处于 `active`
- [ ] `http://127.0.0.1:4173/api/health` 返回 `200`
- [ ] `ros.status` 在 rosbridge 正常时为 `connected`
- [ ] 小车开机后无需输入 Ubuntu 系统密码即可进入桌面会话
- [ ] 开机后不弹出“需要认证 / 登录密钥环未被解锁”
- [ ] 开机后不弹出 NoMachine 更新提示
- [ ] Chromium kiosk 自动打开 `http://127.0.0.1:4173/`
- [ ] 小车断电重启后进入前端登录页，不自动恢复上次已登录页面
- [ ] Chromium 使用独立 kiosk profile，且启动参数包含 `--password-store=basic`
- [ ] 触摸屏用户的 kiosk autostart 文件路径已记录
- [ ] 已记录临时退出 kiosk 和恢复 kiosk 自启动的方法
- [ ] 同一网络电脑可通过 `http://<robot-ip>:4173/` 访问前端
